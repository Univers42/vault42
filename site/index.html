vault42

A self-hostable, zero-knowledge secrets vault. The server stores only ciphertext it can never read — all encryption happens on your machine. multi-tenant ~free

How it works

  • vault42 — the data plane. Stores opaque encrypted envelopes, owner-scoped to your key. It never sees a plaintext or a decryption key.
  • grobase-nano — the contract authority. You register once and it signs a contract binding your key to a tenant. It then idles (scale-to-zero), so the duo costs almost nothing.
  • Your keys live only in your local keystore. Registration sends only your public key. Sharing re-seals for a friend's key — still zero-knowledge.

1 — install & create an identity

cargo install --git https://github.com/Univers42/vault42 vault42-cli
vault42 init                      # generates your local keystore

2 — register yourself

Pick a tenant name and paste your invite token; this page builds the command — run it locally so your key stays on your machine.

vault42 register --authority https://grobase-nano.fly.dev --tenant alice --token …

Then your contract is saved locally and attached to every request.

3 — use your vault

export VAULT42_SERVER=https://vault42.fly.dev
printf 'sk-live-…' | vault42 set prod/stripe   # sealed locally, stored opaque
vault42 get prod/stripe                         # decrypted locally
vault42 ls ; vault42 rotate prod/stripe ; vault42 share prod/stripe --to <address>
